# blackbox criteria: keyrack.source() for test credentials

## usecase.1 = developer runs integration tests

```
given('keyrack is unlocked and keys are set')
  when('developer runs npm run test:integration')
    then('tests execute with credentials injected into process.env')
      sothat('tests can authenticate with external services')

given('keyrack is locked')
  when('developer runs npm run test:integration')
    then('error is thrown with exact unlock command')
      sothat('developer knows how to fix it')
    then('error includes: rhx keyrack unlock --env test --owner ehmpath')

given('keyrack is unlocked but keys are absent')
  when('developer runs npm run test:integration')
    then('error is thrown with exact set commands for each absent key')
      sothat('developer knows which keys to add')
    then('error lists each absent key with: rhx keyrack set --key {KEY} --env test')

given('no keyrack.yml exists in repo')
  when('developer runs npm run test:integration')
    then('tests execute without credential check')
      sothat('repos without keyrack config are unaffected')
```

## usecase.2 = developer runs acceptance tests

```
given('keyrack is unlocked and keys are set')
  when('developer runs npm run test:acceptance:locally')
    then('tests execute with credentials injected into process.env')

given('keyrack is locked')
  when('developer runs npm run test:acceptance:locally')
    then('error is thrown with exact unlock command')

given('keyrack is unlocked but keys are absent')
  when('developer runs npm run test:acceptance:locally')
    then('error is thrown with exact set commands for each absent key')

given('no keyrack.yml exists in repo')
  when('developer runs npm run test:acceptance:locally')
    then('tests execute without credential check')
```

## usecase.3 = ci runs tests

```
given('env vars are set in ci workflow')
  when('ci runs npm run test:integration')
    then('tests execute with credentials from env vars')
      sothat('keyrack prefers env vars over vault')

given('keyrack.yml exists but env vars are set')
  when('ci runs npm run test:integration')
    then('env vars take precedence')
      sothat('ci works without keyrack unlock')
```

## usecase.4 = declapract fix is run on repo with old pattern

```
given('repo has use.apikeys.sh and use.apikeys.json in .agent/')
  when('developer runs declapract fix')
    then('use.apikeys files are removed')
    then('jest.integration.env.ts is updated to use keyrack.source()')
    then('jest.acceptance.env.ts is updated to use keyrack.source()')
    then('package.json test:auth command is removed')
    then('package.json test command no longer sources use.apikeys')

given('repo already has keyrack.source() pattern')
  when('developer runs declapract fix')
    then('no changes are made')
      sothat('idempotent behavior')
```

## usecase.5 = developer opens new terminal mid-workshift

```
given('keyrack was unlocked earlier in the workshift')
  when('developer opens a new terminal')
    when('developer runs npm run test:integration')
      then('tests execute without re-unlock')
        sothat('daemon persists session across terminals')
```

## edge cases

```
given('keyrack.yml declares keys that are not needed by tests')
  when('developer runs npm run test:integration')
    then('all declared keys must be present')
      sothat('explicit declaration is enforced')

given('developer has keys set but keyrack daemon is not started')
  when('developer runs npm run test:integration')
    then('error prompts to unlock')
      sothat('vault access requires explicit unlock each workshift')
```
