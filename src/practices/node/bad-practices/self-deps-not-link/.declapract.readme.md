Self-dependencies must use `link:.` to reference the local package.

A self-dependency occurs when a package depends on itself (e.g., `"my-package": "^1.0.0"` in a package named `my-package`). This can happen accidentally during development or through tooling errors.

Self-dependencies that use version specifiers (like `^1.0.0`) instead of `link:.` will:
- Fetch from npm registry instead of the local code
- Cause unexpected behavior in monorepos
- Lead to circular dependency issues during installation

The fix removes any self-dependencies that are not explicitly `link:.`.
