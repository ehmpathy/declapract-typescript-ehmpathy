## implementation notes

### pattern

match import statements that end with a known error class:
- `UnexpectedCodePathError`
- `BadRequestError`
- `HelpfulError`
- `MalfunctionError`
- `ConstraintError`

### regex

```ts
// match: import { X } from '.../{KnownClass}' or import type { X } from '.../{KnownClass}'
// anchored to line start via ^ with multiline flag (m)
// skips comments, string literals, and mid-line patterns
const KNOWN_CLASSES = ['UnexpectedCodePathError', 'BadRequestError', 'HelpfulError', 'MalfunctionError', 'ConstraintError'];
const IMPORT_PREFIX = `(^\\s*import\\s+(?:type\\s+)?\\{[^}]+\\}\\s+)`;
const FROM_LOCAL_ERROR = `from ['"][^'"]*\\/(${KNOWN_CLASSES.join('|')})['"]`;
const pattern = new RegExp(`${IMPORT_PREFIX}${FROM_LOCAL_ERROR}`, 'gm');
```

### fix

replace the `from '...'` portion with `from 'helpful-errors'`, preserve import prefix via $1 backreference
