# handoff: add array variable support to declapract

## wish

support array variables in `@declapract{variable.X}` expressions so that practices can declare lists (e.g., production approvers) without JSON.parse workarounds.

## why this blocks

the current implementation only stores approvers for `production-on-else` environment. to make the practice reusable across organizations, approvers should come from a declapract variable — but arrays are not supported.

## current limitation

file: `ehmpathy/declapract` → `src/domain.operations/usage/evaluate/projectVariableExpressions/replaceProjectVariablesInDeclaredFileContents.ts`

```typescript
// line 26-27
const flattenedProjectVariables: Record<string, string> = flatten(projectVariables);
```

variables are typed as `Record<string, string>` — only strings. arrays stringify incorrectly.

## proposed solution

1. change type to `Record<string, string | string[]>`

2. detect array on replacement and JSON.stringify:
   ```typescript
   const variableValue = flattenedProjectVariables[variableKey];
   const replacement = Array.isArray(variableValue)
     ? JSON.stringify(variableValue)
     : variableValue;
   return replaceAll(contents, thisVariableExpression, replacement);
   ```

3. usage in practice templates:
   ```typescript
   // getVariables.ts
   productionApprovers: ['uladkasach', 'caseybrookes'],

   // resources.ts — declapract outputs valid JS array literal
   reviewers: { users: @declapract{variable.productionApprovers}, teams: null },
   ```
