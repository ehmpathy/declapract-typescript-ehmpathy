# review.self: has-snap-changes-rationalized (r7)

## deeper reflection — what did the snapshots actually capture?

i read the deleted snapshot file. it captured:

```yaml
secrets:
  ANTHROPIC_API_KEY:
    description: "api key for anthropic api key"
    required: false
  OPENAI_API_KEY:
    description: "api key for openai api key"
    required: false

# ...

env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
```

this was **dynamically generated** by `buildExpectedContent`:
1. read keyrack.yml
2. parse slug to extract key name
3. generate secrets declaration block
4. generate env block with injection

### why this behavior was removed

the firewall handles this at runtime:

| before | after |
|--------|-------|
| declapract generates secrets blocks | template has firewall step |
| keys parsed at build time | keys read at runtime from keyrack.yml |
| secrets injected via YAML | secrets exported to $GITHUB_ENV |

the snapshots captured a build-time transformation that no longer exists.

### should there be new snapshots?

| question | answer |
|----------|--------|
| is there dynamic output? | no — template is static |
| is there behavior to capture? | no — fix returns template unchanged |
| would a snapshot add value? | no — template file is in git |

the old snapshots captured dynamic output. the new behavior has no dynamic output.

### does this delete evidence?

**yes, but intentionally.**

the evidence shows what the OLD behavior produced. the OLD behavior is removed. retained snapshots of removed behavior would create confusion:
- future developers might think we still generate these blocks
- the snapshots would be orphaned (no test references them)
- they would drift from reality

the NEW behavior produces different output (template with firewall step). this output is visible in:
- the template file itself
- the git diff in this PR
- the test assertion (`result.contents === templateContent`)

## found issues

none.

## why it holds

### the snapshots captured removed behavior

the snapshots showed secrets blocks generated by buildExpectedContent. that function was deleted. the behavior moved to firewall (runtime). the snapshots are not just orphaned — they document a past that no longer applies.

### no dynamic output to snapshot

the new behavior is:
```typescript
fix: (contents, context) => ({ contents: context.declaredFileContents })
```

the output IS the template. the template IS the snapshot. there is no transformation to capture.

### the PR diff serves as the snapshot

reviewers see:
- **before**: template with secrets parse logic references
- **after**: template with firewall step

this diff IS the vibecheck. a snapshot would duplicate the template file.

### conclusion

deleted snapshots captured removed behavior. no new snapshots needed because the new behavior is declarative (template = output). the template file in git serves as the snapshot.
