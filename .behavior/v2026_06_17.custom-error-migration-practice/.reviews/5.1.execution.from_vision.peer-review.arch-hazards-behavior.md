# blocker.1: Regex-based import detection and replacement is non-deterministic and fragile

**rule**: rule.forbid.behavior-hazards.md

**locations**:
- src/practices/errors/bad-practices/local-error-imports/src/**/*.ts.declapract.ts

The check and fix functions use a custom regex (with ^, \s+, [^'"]* and class-name-ending assumptions) to parse import statements. This does not behave predictably under all conditions:
- Multiline imports (import {\n  X\n} from 'local/X') do not match because prefix expects }\s+from on same line.
- Imports without whitespace between } and from (import {X}from 'local/X') fail due to \s+ requirement.
- Paths with extensions (.../UnexpectedCodePathError.ts) fail because class name must be immediately followed by ['"].
- Imports of non-matching symbols from a matching path (import { Foo } from '.../UnexpectedCodePathError') are incorrectly rewritten, corrupting code.
This creates hidden side effects, unexpected defaults on malformed-but-valid imports, and silent data (source) corruption. Matches the rule's definition of behavior hazards: intermittent detection, passes some tests but fails on real files, order/format dependent.

**snippet**:
```ts
const IMPORT_PREFIX = `(^\\s*import\\s+(?:type\\s+)?\\{[^}]+\\}\\s+)`;
const FROM_LOCAL_ERROR = `from ['"][^'"]*\\/(${KNOWN_CLASSES.join('|')})['"]`;
const LOCAL_ERROR_IMPORT_PATTERN_SOURCE = `${IMPORT_PREFIX}${FROM_LOCAL_ERROR}`;

const LOCAL_ERROR_IMPORT_PATTERN = new RegExp(
  LOCAL_ERROR_IMPORT_PATTERN_SOURCE,
  'm',
);

const LOCAL_ERROR_IMPORT_PATTERN_GLOBAL = new RegExp(
  LOCAL_ERROR_IMPORT_PATTERN_SOURCE,
  'gm',
);

export const check: FileCheckFunction = (contents) => {
  if (contents && LOCAL_ERROR_IMPORT_PATTERN.test(contents)) return;
  UnexpectedCodePathError.throw(
    'no local error imports found; file uses helpful-errors or custom errors',
    { hint: 'this file does not need migration to helpful-errors' },
  );
};

export const fix: FileFixFunction = (contents) => {
  if (!contents) return {};
  return {
    contents: contents.replace(
      LOCAL_ERROR_IMPORT_PATTERN_GLOBAL,
      "$1from 'helpful-errors'",
    ),
  };
};
```
