# playtest: keyrack.source() for test credentials

## prerequisites

- this repo: declapract-typescript-ehmpathy
- cd to repo root directory
- rhachet >=1.39.1 installed globally
- keyrack unlocked: `rhx keyrack unlock --env test --owner ehmpath`

## sandbox

all test file operations target `.temp/` or operate on this repo's test suite.

---

## playtest.1: bad-practice detection and fix

this behavior adds a bad-practice to detect and remove legacy use.apikeys files.

### step 1: verify bad-practice files exist

```sh
find src/practices/cicd-common/bad-practices/old-use-apikeys -name "*.declapract*.ts" | wc -l
```

**expect:** 4 files (2 for .sh, 2 for .json — each has .declapract.ts and .declapract.test.ts)

### step 2: verify bad-practice check returns EXISTS

```sh
grep -A3 'export const check' src/practices/cicd-common/bad-practices/old-use-apikeys/.agent/repo=.this/role=any/skills/use.apikeys.sh.declapract.ts
```

**expect:**
```
export const check = FileCheckType.EXISTS;
```

### step 3: verify bad-practice fix returns null contents

```sh
grep -A3 'export const fix' src/practices/cicd-common/bad-practices/old-use-apikeys/.agent/repo=.this/role=any/skills/use.apikeys.sh.declapract.ts
```

**expect:**
```
export const fix: FileFixFunction = () => {
  return { contents: null };
};
```

**note:** use.apikeys.json.declapract.ts has identical check/fix logic.

## pass/fail criteria

- ✓ pass if:
  - bad-practice files exist with correct check/fix logic
  - jest env files contain keyrack.source() call
  - keyrack.source() is guarded by existence check
  - all tests pass
  - old use.apikeys files are removed from best-practice

- ✗ fail if:
  - bad-practice check is not FileCheckType.EXISTS
  - bad-practice fix does not return `{ contents: null }`
  - keyrack.source() is not called in jest env files
  - keyrack.source() is not guarded by existence check
  - any tests fail
  - old use.apikeys files still exist in best-practice
