## pattern 3: readUseApikeysConfig.declapract.test.ts

**location**: `src/utils/readUseApikeysConfig.declapract.test.ts`

**citation [3]**:
```typescript
describe('readUseApikeysConfig', () => {
  given('a project root directory', () => {
    when('use.apikeys.json exists with valid content', () => {
      then('it should return the parsed config', async () => {
        const tempDir = await util.promisify(fs.mkdtemp)(
          path.join(os.tmpdir(), 'readUseApikeysConfig-'),
        );
        const configDir = path.join(
          tempDir,
          '.agent/repo=.this/role=any/skills',
        );
        await util.promisify(fs.mkdir)(configDir, { recursive: true });
        const configPath = path.join(configDir, 'use.apikeys.json');
        await util.promisify(fs.writeFile)(
          configPath,
          JSON.stringify({
            apikeys: {
              required: ['ANTHROPIC_API_KEY', 'OPENAI_API_KEY'],
            },
          }),
        );

        const result = await readUseApikeysConfig({
          projectRootDirectory: tempDir,
        });

        expect(result).toEqual({
          apikeys: {
            required: ['ANTHROPIC_API_KEY', 'OPENAI_API_KEY'],
          },
        });
      });
    });
  });
});
```

**relation**: [REPLACE] — will be replaced with readKeyrackConfig.test.ts that tests `.agent/keyrack.yml` parse logic.

## pattern 4: .test.yml.declapract.test.ts apikeys tests

**location**: `src/practices/cicd-common/best-practice/.github/workflows/.test.yml.declapract.test.ts`

**citation [4]**:
```typescript
describe('.test.yml.declapract buildExpectedContent', () => {
  given('a template with workflow_call inputs', () => {
    when('one apikey is required', () => {
      then('it should add secrets declaration to workflow_call', () => {
        const result = buildExpectedContent({
          template,
          apikeys: ['ANTHROPIC_API_KEY'],
        });

        expect(result).toContain('secrets:');
        expect(result).toContain('ANTHROPIC_API_KEY:');
        expect(result).toContain(
          'description: "api key for anthropic api key"',
        );
      });

      then('it should add env block to test-integration job', () => {
        const result = buildExpectedContent({
          template,
          apikeys: ['ANTHROPIC_API_KEY'],
        });

        expect(result).toContain('env:');
        expect(result).toContain(
          'ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}',
        );
      });
    });
  });
});
```

**relation**: [EXTEND] — tests will be updated to pass keyrack config shape instead of apikeys array. the test structure remains the same, but input changes from `apikeys: ['KEY']` to read from keyrack.yml format.

## test coverage needed for new code

new tests required for keyrack.source() pattern:

1. **readKeyrackConfig tests**
   - keyrack.yml exists with valid content → returns parsed config
   - keyrack.yml does not exist → returns null
   - keyrack.yml contains malformed yaml → returns null gracefully

2. **.test.yml secrets injection tests**
   - keyrack.yml with env.test keys → adds secrets declaration
   - keyrack.yml without env.test → returns template unchanged
   - multiple keys in env.test → adds all secrets

3. **jest env file tests** (if needed)
   - keyrack.yml exists → keyrack.source() is called
   - keyrack.yml does not exist → no-op
