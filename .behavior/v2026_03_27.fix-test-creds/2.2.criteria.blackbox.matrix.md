## matrix 2: ci test execution

dimensions:
- env vars: set | not set
- keyrack.yml: exists | absent

| ind: env vars | ind: keyrack.yml | dep: outcome | dep: credential source |
|---------------|------------------|--------------|----------------------|
| set | exists | tests run | env vars (precedence) |
| set | absent | tests run | env vars |
| not set | exists | **gap** | **undefined** |
| not set | absent | tests run | no credentials needed |

### gap found

**row 3**: when env vars not set but keyrack.yml exists in ci, what happens?

options:
1. ci should always have env vars set (documentation)
2. keyrack.source() fails with unlock error (current behavior)
3. ci has its own keyrack unlock mechanism

**recommendation**: document that ci must set env vars. keyrack.source() will fail otherwise. this is intentional - ci shouldn't need interactive unlock.
