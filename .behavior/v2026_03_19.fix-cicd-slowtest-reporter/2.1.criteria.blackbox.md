# blackbox criteria: slowtest reporter best practice

## episode experience

# usecase.1 = spot slow integration tests locally
given(a project with declapract-typescript-ehmpathy best practices applied)
  when(developer runs integration tests via `npm run test:integration`)
    then(a slowtest report appears in terminal after test completion)
      sothat(developer immediately sees which tests exceeded the threshold)
    then(report shows file paths, describe blocks, and durations in tree format)
      sothat(developer knows exactly where to look to optimize)
    then(slow tests are marked with visual indicator)
      sothat(offenders stand out from normal tests)

# usecase.2 = track slow tests over time
given(a project with declapract-typescript-ehmpathy best practices applied)
  when(developer runs integration tests)
    then(json report is written to `.slowtest/integration.report.json`)
      sothat(ci systems can consume structured data for optimization)
  when(developer runs `git status` after tests)
    then(the `.slowtest/integration.report.json` file is ignored)
      sothat(report files are not accidentally committed)

# usecase.3 = receive slowtest reporter via declapract upgrade
given(a project with declapract-typescript-ehmpathy)
  when(developer runs `npx declapract apply`)
    then(jest.integration.config.ts includes slowtest reporter configuration)
      sothat(slow test visibility is automatic without manual setup)
    then(.gitignore includes `.slowtest/integration.report.json` entry)
      sothat(report files are excluded from version control)

## exchange experience

# exchange.1 = slowtest terminal output
given(integration tests have completed)
  when(any test exceeds 10s threshold)
    then(terminal shows report with tree structure)
    then(file path appears with total duration)
    then(nested describe/given/when/then blocks show individual durations)
  when(no tests exceed threshold)
    then(no slowtest report appears)
      sothat(output stays clean when all tests are fast)

# exchange.2 = json report file
given(integration tests have completed)
  when(tests finish)
    then(`.slowtest/integration.report.json` is created or updated)
    then(json contains structured test duration data)
      sothat(external tools can parse and analyze)
