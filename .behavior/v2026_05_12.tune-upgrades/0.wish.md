# handoff: declapract-typescript-ehmpathy upgrade defects

## context

upgrade from `declapract-typescript-ehmpathy@0.36.3` to `@0.47.69` surfaced several undocumented variables and inconsistent environment names.

## defects found

### 1. inconsistent environment names: `dev` vs `prep`

some practices reference `*.dev`, others reference `*.prep`. this forces repos to define both:

```yaml
# required to satisfy all practices
awsAccountId:
  dev: 874711128849
  prep: 874711128849  # same value, different key
  prod: 398838478359

databaseTunnelHost:
  dev: aws.ssmproxy.ahbodedb.dev
  prep: aws.ssmproxy.ahbodedb.prep  # same pattern, different key
  prod: aws.ssmproxy.ahbodedb.prod

databaseClusterHost:
  dev: ahbodedb.cluster-cvpnwcrisal7.us-east-1.rds.amazonaws.com
  prep: ahbodedb.cluster-cvpnwcrisal7.us-east-1.rds.amazonaws.com  # duplicate
  prod: ahbodedb.cluster-cmky0uvtwzbh.us-east-1.rds.amazonaws.com
```

**recommendation**: standardize on one environment name (`prep` preferred per `prefer.env_access.prep_over_dev`). update all practices to use consistent environment keys.

### 2. undocumented variable: `databaseTunnelHost`

the `persist-with-rds` practice requires `databaseTunnelHost` but this is not documented in the practice readme or changelog.

**error seen**:
```
Variable was declared in file contents but its value was not defined: 'databaseTunnelHost.prep'
```

**where used**:
- `src/practices/persist-with-rds/best-practice/config/prep.json` (line 19)
- `src/practices/persist-with-rds/best-practice/config/prod.json`

**recommendation**: add `databaseTunnelHost` to the practice readme as a required variable with format and example values.

### 3. undocumented variable: `reviewers.users`

the `provision-github` practice requires `reviewers.users` but this is not documented.

**error seen**:
```
Variable was declared in file contents but its value was not defined: 'reviewers.users'
```

**where used**:
- `src/practices/provision-github/best-practice/provision/github.repo/resources.ts` (line 137)
- used for `production-on-else` environment reviewers

**recommendation**: add `reviewers.users` to the practice readme as a required variable with format example:

```yaml
reviewers:
  users: ['githubusername1', 'githubusername2']
```

## impact

repos that upgrade from older versions will fail silently until they discover these undocumented variables through trial and error.

## suggested improvements

1. **document all required variables** in each practice's readme
2. **standardize environment names** to `prep` and `prod` (or `test`, `prep`, `prod`)
3. **add validation** that fails early with clear error message if required variables are absent
4. **include upgrade guide** in changelog for incompatible changes

## affected repos

all ahbode repos that use `persist-with-rds` or `provision-github` practices and upgrade to `declapract-typescript-ehmpathy` >= 0.47.x.
