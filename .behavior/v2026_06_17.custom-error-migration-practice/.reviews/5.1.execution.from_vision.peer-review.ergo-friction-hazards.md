# blocker.1: error messages and outputs not snapped

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/practices/errors/bad-practices/local-error-imports/src/**/*.ts.declapract.test.ts
- src/practices/errors/bad-practices/local-error-imports/src/**/*.ts.declapract.test.ts
- src/practices/errors/bad-practices/local-error-imports/src/**/*.ts.declapract.test.ts

The check tests catch UnexpectedCodePathError and only do expect(error.message).toContain(...) without .toMatchSnapshot(). Fix tests use .toBe() or .toContain() on the resulting contents instead of snapshots. This means error messages and fixed code (the contracts/output the declapract process and humans see) are not verified with snapshots. Per the rule, every error path and output variant must be snapped for reviewers to see the actual experience, ensure consistency, and catch drift. Unsnapped friction paths will ship broken or unclear.

**snippet**:
```ts
      const error = await getError(() => check(contents, {} as any));
      expect(error).toBeInstanceOf(UnexpectedCodePathError);
      expect(error.message).toContain('no local error imports found');
```

# nitpick.1: loose partial assertions on fix outputs

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/practices/errors/bad-practices/local-error-imports/src/**/*.ts.declapract.test.ts

Some fix tests use expect(fixed).toContain(...) + .not.toContain(...) instead of full exact match or snapshot. This provides weaker verification of the exact output the user/declapract will see and can miss unrelated changes in the fixed code.

**snippet**:
```ts
      const { contents: fixed } = await fix(contents, {} as any);

      expect(fixed).toContain("from 'helpful-errors'");
      expect(fixed).not.toContain('../../utils/errors/');
```
