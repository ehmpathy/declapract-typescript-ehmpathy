# evaluation: keyrack.source() for test credentials

## summary (as implemented)

replaced legacy `use.apikeys.sh` + `use.apikeys.json` pattern with `keyrack.source()` for test credential injection in declapract best practices.

**implemented:**
1. deleted use.apikeys.sh and use.apikeys.json from cicd-common best-practice
2. updated jest.integration.env.ts and jest.acceptance.env.ts to use keyrack.source()
3. updated package.json to remove test:auth command
4. created bad-practice to migrate repos with old pattern
5. updated buildWorkflowSecretsBlock to invoke keyrack sdk for required vars
6. updated .test.yml workflow to run prepare:rhachet before tests
7. created withKeyrackContext test fixture for buildWorkflowSecretsBlock tests
