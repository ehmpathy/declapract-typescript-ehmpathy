# blueprint: fix acceptance test dotmove

## summary

reproduce and fix the bug where `declapract fix` drops files in hidden directories when it moves files from `accept.blackbox/` to `blackbox/`.

## root cause analysis

the current bad-practice definition uses these glob patterns:
- `accept.blackbox/**/*.ts.declapract.ts`
- `accept.blackbox/**/*.acceptance.test.ts.declapract.ts`

these patterns have two issues:
1. **file extension filter**: only match `.ts` files (not `.json`, `.yml`, `.md`, etc.)
2. **hidden path exclusion**: glob implementations often exclude dotfiles/dotdirs by default unless configured with `dot: true`

the fix function itself is correct — it just never receives the files to fix.

### catch-all fix contract

```ts
// src/practices/tests/bad-practices/old-acceptance-dir-location/accept.blackbox/**/*.declapract.ts

import { FileCheckType, type FileFixFunction } from 'declapract';

export const check = FileCheckType.EXISTS;

/**
 * .what = moves any file from accept.blackbox/ to blackbox/
 * .why = ensures all files are moved, not just .ts files — prevents silent data loss
 */
export const fix: FileFixFunction = (contents, context) => {
  const newPath = context.relativeFilePath.replace(
    /^accept\.blackbox\//,
    'blackbox/',
  );
  return {
    contents: contents ?? null,
    relativeFilePath: newPath,
  };
};
```

### rule brief contract

```markdown
# rule.require.glob-includes-dotpaths

## .what

glob patterns in declapract practices must explicitly account for hidden paths (dotfiles and dotdirs)

## .why

- glob implementations often exclude dotfiles/dotdirs by default
- silent exclusion leads to data loss when practices run fix operations
- users expect "move all files" to mean *all* files, not "all visible files"

## .how

when a practice needs to match files in a directory tree:

1. use catch-all patterns (`**/*`) rather than extension-specific patterns (`**/*.ts`)
2. verify glob behavior with hidden paths in integration tests
3. if the glob library supports it, enable `dot: true` or equivalent

## .examples

### bad — extension-specific, may miss hidden paths

```
accept.blackbox/**/*.ts.declapract.ts
```

### good — catch-all covers all file types

```
accept.blackbox/**/*.declapract.ts
```

### good — explicit test for hidden paths

```ts
it('should match files in hidden directories', async () => {
  const files = glob('accept.blackbox/**/*', { dot: true });
  expect(files).toContain('accept.blackbox/.test/assets/.rhachet/manifest.json');
});
```

## .see also

- `old-acceptance-dir-location` bad practice — the source of this lesson
```

## execution order

1. **reproduce** — write integration test that fails with current behavior
2. **fix** — add catch-all glob pattern `accept.blackbox/**/*.declapract.ts`
3. **verify** — run integration test, confirm it passes
4. **document** — create rule brief at `.agent/repo=.this/role=any/briefs/rule.require.glob-includes-dotpaths.md`
5. **validate** — run full test suite to ensure no regressions

## risks

- the catch-all pattern may match files that should not be moved (e.g., `.git/` contents)
  - mitigation: declapract should already exclude `.git/` — verify this
- glob pattern match order may cause conflicts
  - mitigation: test that more-specific patterns take precedence

## open questions
