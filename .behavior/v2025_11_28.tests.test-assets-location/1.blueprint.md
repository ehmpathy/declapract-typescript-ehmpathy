blueprint = bad-practice to detect and rename `__test_utils__` and `__test_assets__` to `.test/assets/`

---

location: `src/practices/tests/bad-practices/old-test-utils-and-assets-location/`

---

files:

1. `.declapract.readme.md`
   - documents the practice and why it exists

2. `**/__test_utils__/**/*.declapract.ts`
   - detects files in `__test_utils__` directories via `FileCheckType.EXISTS`
   - renames directory to `.test/assets/` by returning new `relativeFilePath`
   - preserves directory structure after `__test_utils__/`

3. `**/__test_assets__/**/*.declapract.ts`
   - detects files in `__test_assets__` directories via `FileCheckType.EXISTS`
   - renames directory to `.test/assets/` by returning new `relativeFilePath`
   - preserves directory structure after `__test_assets__/`

4. `**/*.ts.declapract.ts`
   - detects `.ts` files that import from `__test_utils__` or `__test_assets__`
   - replaces import paths to use `.test/assets/` instead
   - simple string replacement of directory name in import paths

---

key implementation details:

- file movement uses `relativeFilePath` in fix return object
- path transformation: `any/path/__test_utils__/foo/bar.ts` → `any/path/.test/assets/foo/bar.ts`
- import path fix: `from '../__test_utils__/foo'` → `from '../.test/assets/foo'`
