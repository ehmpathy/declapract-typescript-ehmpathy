wish = add bad-practice to migrate custom error imports to helpful-errors

## context

when declapract upgrades a repo, the `errors` practice adds `helpful-errors` to package.json and provides a fix for `@ehmpathy/error-fns` imports. however, repos with custom error implementations (e.g., `src/utils/errors/UnexpectedCodePathError.ts`) have their files deleted but imports that point to those files are not updated.

## example

before upgrade:
```ts
// src/logic/doStuff.test.ts
import { UnexpectedCodePathError } from '../../utils/errors/UnexpectedCodePathError';
```

after upgrade:
- file `src/utils/errors/UnexpectedCodePathError.ts` is deleted
- import still points to deleted file → type error

## desired behavior

add a bad-practice that:
1. detects imports from local error paths (e.g., `@src/utils/errors/*`, `../../utils/errors/*`)
2. replaces them with imports from `helpful-errors`

## scope

- target: `src/**/*.ts` (includes test files)
- pattern: imports of common error classes (`UnexpectedCodePathError`, `BadRequestError`, `HelpfulError`, etc.) from local paths
- action: replace with `import { X } from 'helpful-errors'`
