# 5.1 execution — fix: mask-timestamps husky guard

## decisions locked (from vision + wisher 2026-06-25)

- **scope**: default deny — block all staged files *except* `.ts`/`.sh`
- **regex**: `T?HH:MM:SS{tz}` — lead `T` optional (bare `[HH:MM:SS]` matches), accept common ISO tz forms (`Z`, `±HH:MM`, `±HHMM`, none)
- **placement**: `src/practices/husky/best-practice/.husky/check.timestamps.sh`, sourced from the template `pre-commit`; mirror to this repo's root `.husky/`
- **output**: minimal posix shell + emoji; halt with `✋` + verbatim message `timestamps are forbidden in snapshots. mask them to prevent permadrift`
- **exit code**: `2` (constraint)
- **dogfood self-block accepted (NO `.behavior/` exception)**: an earlier draft skipped `.behavior/*` to let the repo commit its own route docs. peer review (behavior-intent-coverage) correctly rejected that as an **unauthorized preemptive escape hatch** — the wisher chose pure default-deny and left the escape hatch (q2) open. the `.behavior/` skip was **removed**. consequence: the repo's own timestamped route docs are self-blocked until q2 (escape hatch) is decided by the wisher. this is the *accepted* dogfood consequence, not a defect.

## implementation notes

### files
- `src/practices/husky/best-practice/.husky/check.timestamps.sh` — the guard (new)
- `src/practices/husky/best-practice/.husky/pre-commit` — sources the guard after `check.yalc.sh`
- `.husky/check.timestamps.sh` — root mirror (identical, so declapract EQUALS stays consistent)
- `.husky/pre-commit` — root mirror sources the guard

### design choices
- **here-doc, not pipe**: the scan loop reads staged files via a here-doc (`done <<EOF`) so it runs in the *current* shell — `exit 2` halts the commit directly on a match, and a clean pass falls through (lets any later-sourced checks run). a pipe would run the loop in a subshell and need fragile exit-code propagation.
- **staged content scanned** via `git show ":$file"` (not the on-disk copy) — correct pre-commit scope; immune to partial-stage drift.
- **fail loud on git read failure** (peer-review fix): `git show` is captured with `2>&1` and guarded by `if ! …`; on failure the guard prints the git error and exits `1` (malfunction). this replaces the earlier `2>/dev/null`, which silently swallowed errors (a failhide).
- **deletions skipped** via `--diff-filter=ACM`.
- **filenames with spaces** handled via `IFS= read -r` (more robust than the peer checks).

### regex
`T?[0-9][0-9]:[0-9][0-9]:[0-9][0-9](Z|[+-][0-9][0-9]:?[0-9][0-9])?`
- optional lead `T`; `HH:MM:SS`; optional tz (`Z`, `±HH:MM`, `±HHMM`)
- verified via grepsafe: matches `…T[HH:MM:SS]Z`, `…+05:00`, `…+0500`, bare `[HH:MM:SS]`, `T[HH:MM:SS]`; rejects date-only, `HH:MM`, version strings, shas, words.

### no `.behavior/` exception (peer-review fix)
an earlier draft skipped `.behavior/*` so the repo could commit its own route docs. peer review rejected that as an **unauthorized preemptive escape hatch** — the wisher chose pure default-deny and left the escape hatch (q2) open. the skip was **removed**. the repo's own timestamped route docs are now self-blocked until the wisher decides q2; this is the *accepted* dogfood consequence per the vision, not a defect.
