# vision: fix acceptance test dotmove

## the outcome world

### before

when a repo has the old `accept.blackbox/` directory structure with nested hidden directories:

```
accept.blackbox/
├── .test/
│   └── assets/
│       └── with-keyrack-manifest/
│           ├── .rhachet/
│           │   └── keyrack.manifest.json
│           └── .agent/
│               └── keyrack.yml
├── somefile.ts
└── another.acceptance.test.ts
```

after `declapract fix` runs:

```
blackbox/
├── somefile.ts            ✅ moved
└── another.acceptance.test.ts  ✅ moved

# but the hidden dirs are silently dropped!
# .test/assets/with-keyrack-manifest/.rhachet/keyrack.manifest.json  ❌ gone
# .test/assets/with-keyrack-manifest/.agent/keyrack.yml              ❌ gone
```

### after (goal)

all files are moved, including those in hidden directories:

```
blackbox/
├── .test/
│   └── assets/
│       └── with-keyrack-manifest/
│           ├── .rhachet/
│           │   └── keyrack.manifest.json  ✅ moved
│           └── .agent/
│               └── keyrack.yml            ✅ moved
├── somefile.ts            ✅ moved
└── another.acceptance.test.ts  ✅ moved
```

### the "aha" moment

- no more silent data loss when declapract practices apply
- hidden test assets (fixtures, manifests, agent configs) are preserved
- users can trust that `declapract fix` moves all files, not just visible ones

## user experience

### usecase

1. user has a repo with `accept.blackbox/` directory that contains test assets
2. some assets live in hidden directories (`.test/`, `.rhachet/`, `.agent/`)
3. user runs `npx declapract fix` to upgrade to best practices
4. **expectation**: all files move from `accept.blackbox/` to `blackbox/`
5. **reality (bug)**: hidden directory contents are silently dropped

### timeline

1. user applies declapract fix
2. user commits changes (without notice of the loss)
3. tests start to fail due to absent fixtures
4. user discovers the hidden files were never moved
5. user must manually recover files from git history

## mental model

> "declapract moves my acceptance tests from `accept.blackbox/` to `blackbox/`"

users assume "moves" means *all* files. the current behavior violates this mental model by silent exclusion of hidden paths.

## evaluation

### root cause hypothesis

the glob patterns in the bad-practice definition:
- `accept.blackbox/**/*.ts`
- `accept.blackbox/**/*.acceptance.test.ts`

these patterns:
1. only match `.ts` files (not `.json`, `.yml`, etc.)
2. may not traverse hidden directories, dependent on glob implementation

### pros of the fix

- no silent data loss
- predictable behavior that matches user expectations
- test fixtures and hidden configs are preserved

### cons / considerations

- need to understand if this is a declapract glob limitation or a practice definition issue
- may need additional glob patterns or a more inclusive pattern

### edgecases

- hidden directories at multiple nested levels
- mixed file types in hidden directories (`.json`, `.yml`, `.md`)
- hidden files (not just hidden directories)
- symlinks to hidden directories

## next steps

1. write an integration test that reproduces the bug
   - use `genTempDir` to create a temp directory
   - populate it with `accept.blackbox/.test/assets/.../.rhachet/file.json`
   - apply the bad-practice fix
   - verify which files moved vs dropped

2. bisect to find root cause
   - is it the glob pattern definition?
   - is it declapract's glob behavior?
   - is it rhachet's glob behavior (if rhachet is in use)?

3. fix the root cause and add regression tests

4. crystallize the lesson learned into a rule brief
   - create `.agent/repo=.this/role=any/briefs/rule.require.glob-includes-dotpaths.md`
   - document that glob patterns in declapract practices must explicitly include hidden paths
   - provide examples of correct vs incorrect patterns
   - ensure all future practices account for dotfiles and dotdirs

5. scan repo for other violations (apply the rule)
   - this bug is sneaky — it may lurk in other practices with extension-specific patterns
   - search for all `**/*.ts.declapract.ts` patterns that move/delete files
   - audit each practice against the rule brief
   - fix all identified violations with catch-all patterns
